# Roammand 品牌设计规范

[English](README.md) · **简体中文**

**产品定位：**隐私优先、无需账号，用移动设备控制你自己的电脑。

**品牌口号：**离开桌面，工作仍在继续。

**英文：**Leave the desk. Keep work moving.

Roammand 将个人工作空间从桌面延伸到移动端。身份与控制权始终属于用户，进行中的工作随时触手可及。远程控制是起点，更连续、更具自主性的个人工作流是方向。

## 名称与语气

“Roammand” 由 *roam* 与 *command* 组合而来。拼写和大小写必须保持为 `Roammand`，不拆分、不翻译。

- **直接：**先说明操作、设备或状态。
- **克制：**语气平静、自信，不夸大。
- **个人：**始终围绕用户自己的电脑、工作与控制权。
- **准确：**在必要时明确 Controller、恢复时间、权限和数据边界。

## 夜极光色彩系统

夜墨色构成工作空间，极光靛建立品牌识别，信号青标记连接与活动。明亮的反色表面为主操作和 Logo 提供清晰对比。

| 角色 | 色值 | 用途 |
| --- | --- | --- |
| 画布 | `#090B1F` | App 背景与图标底色 |
| 深色表面 | `#11142D` | 导航和默认卡片 |
| 抬升表面 | `#191E41` | 对话框和强调面板 |
| 极光靛 | `#7E7BFF` | 品牌焦点与选中状态 |
| 柔光靛 | `#A9A7FF` | 次级品牌文字和图标 |
| 信号青 | `#32C9F3` | 实时连接和进度 |
| 反色表面 | `#F3F2FF` | 主操作和 Logo 对比色 |
| 主要文字 | `#F3F2FF` | 深色表面上的主要内容 |
| 次要文字 | `#B8BBD6` | 辅助内容 |
| 轮廓 | `#34395F` | 分隔线和弱边框 |
| 在线 | `#59D8B5` | 就绪和已连接 |
| 注意 | `#F6C66B` | 需要权限或用户操作 |
| 紧急 | `#FF728B` | 失败、撤销或停止控制 |

不要把靛青渐变当作语义状态色。绿色表示就绪，琥珀色表示需要注意，红色只用于破坏性操作或停止状态。正文和关键控件必须达到 WCAG AA 对比度。

## 字体与布局

Roammand 使用各平台的系统无衬线字体，保持原生渲染和一致的可读性。标题使用 600–700 字重并略微收紧字距；正文使用舒展的 `1.45` 行高。

- 使用 4 点间距网格，常用级别为 4、8、12、16、20、24、32 和 48。
- 主要触控区域至少为 48 点，只有图标的控件至少为 44 点。
- 卡片和对话框使用 20 点圆角，控件和输入框使用 14 点圆角；完整胶囊形只用于简短状态标签。
- 优先使用安静的 1 点轮廓线，避免装饰性阴影。层级只应用于解释信息结构，不应装饰所有表面。
- 宽桌面布局应限制阅读区域宽度；窄窗口中允许导航从侧边栏切换为底部栏。
- 正确处理安全区域、横屏、键盘遮挡、文字缩放和减少动态效果设置。

动效用于确认状态变化：本地反馈约 160 ms，布局过渡约 240 ms。避免循环装饰、远程画面视差，以及任何会延迟“停止”或“紧急停止”的动画。

## Logo 系统

Logo 以一条控制路径连接桌面工作空间与移动 Controller。双框轮廓和控制点在小尺寸下仍保持清晰识别。

| 资产 | 用途 |
| --- | --- |
| `roammand-app-icon.svg` | 各平台 App Icon 的不透明源文件 |
| `roammand-mark.svg` | 夜极光表面上的主标志 |
| `roammand-mark-monochrome.svg` | 单色和反色场景 |
| `roammand-tray-template.svg` | 菜单栏和系统托盘小尺寸源文件 |
| `roammand-app-icon-1024.png` | 用于审阅的 1024 px 位图母版 |

Logo 周围的安全空间至少等于控制圆环的直径。保持其方向、几何和色彩关系；单色托盘资产遵循各平台模板图标规则。

在 macOS 上运行以下命令，重新生成提交到仓库中的 PNG 和 ICO 衍生资产：

```bash
./scripts/render_brand_assets_macos.sh
```

确定性生成脚本需要 Quick Look 和 Python Pillow。日常应用构建直接使用已提交的衍生资产，不依赖这些工具。

## 产品资产检查清单

公开发行前，应在真实主屏幕、Dock/任务栏、Spotlight/开始菜单、安装器界面以及 16 px 托盘/菜单栏中检查 App Icon。商店截图和宣传素材必须使用与实际发行版本一致的产品界面，保留安全区域，并避免显示真实设备身份、连接地址、配对码或诊断信息。
