# Security Policy

## Support scope

Security fixes target the current public source tree and the releases listed on GitHub. Versions outside that list should be upgraded before assessment.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub private vulnerability reporting.

Do not include real private keys, pairing secrets, authentication transcripts, SDP, ICE candidates, typed text, pointer coordinates, or credentials in a report. Use synthetic data and provide the smallest reproducible case.
