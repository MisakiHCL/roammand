# Roammand 用户指南

[English](README.md) · **简体中文**

> 当前面向公众提供的通用安装包只有 macOS 版本。移动 Controller 需要自行从
> 源码构建，或使用维护者另行提供的测试版本；暂无公开 App Store 或 Android
> 安装包下载。

## 开始之前

- 当前 macOS 版本要求 macOS 14.4 或更高版本。
- Mac 必须授予 Roammand“屏幕录制”和“辅助功能”权限，之后才能接受连接。
- 一台 Host 同时只接受一个传入 Controller 会话。
- iOS 源码目标要求 iOS / iPadOS 13 或更高版本，当前界面固定为横屏；Android
  源码目标要求 Android 7.0 / API 24 或更高版本。
- 当前版本只尝试 ICE 直连，没有 TURN 中继兜底。某些严格或对称的网络组合可能
  无法连接。

## 安装并授权 Mac

1. 从 [GitHub Releases](https://github.com/MisakiHCL/roammand/releases/latest)
   下载经过签名和公证的 `Roammand.pkg`。
2. 打开安装包，并批准管理员提示，以安装 App 和所需的 Host 组件。
3. 启动 Roammand，打开“**此电脑**”。
4. 分别点击两项“**去设置**”，在系统设置中授予“屏幕录制”和“辅助功能”权限。
5. 返回 Roammand。只有两项权限都就绪后，配对和传入连接才会变为可用。

权限请求只会由这些本机设置操作发起。远程连接不会代替 Mac 用户请求权限。

## 准备 Controller

如果使用另一台 Mac 作为 Controller，可安装同一公开安装包并打开“**我的电脑**”；
只有需要接受传入控制的电脑才需要配置 Host 权限。

当前没有公开的移动应用商店版本。源码构建者应使用[从源码构建
Roammand](../BUILDING.zh-CN.md)中的实体设备命令。如果维护者另行提供了测试版本，
请只通过邀请中指定的渠道安装，不要使用非官方镜像。

首次启动时设置设备名称。Host 请求本机批准时会显示这个有长度限制的名称；它只是
便于识别的标签，不是身份凭证。

## 使用二维码配对手机或平板

1. 在 Mac 上打开“**此电脑**”，创建手机二维码邀请。
2. 在移动 Controller 中打开扫描器并扫描实时二维码。不支持从相册、文件、剪贴板
   导入，也不支持手工输入二维码文本。
3. 检查 Mac 上显示的 Controller 名称，并在本机批准。
4. 在“**我的电脑**”中选择这台 Mac，然后点击“**连接**”。

二维码配对不使用四词比对；它依赖实时、经过认证的邀请，同时仍必须由 Host 本机
批准。

## 使用配对码配对另一台电脑

1. 在 Host 的“**此电脑**”中创建桌面配对码。
2. 在 Controller 的“**我的电脑**”中输入该配对码。
3. 核对两台电脑显示的全部四个英文验证词；任意一词不一致都应取消。
4. 在 Host 本机批准名称正确的 Controller，然后从已保存的电脑卡片发起连接。

二维码邀请与桌面配对码最多两分钟后过期。配对授权是单向的；如果要交换 Host 与
Controller 的方向，需要重新配对并单独批准。

Roammand 会优先使用 Mac 的系统电脑名称。如果多台电脑名称相似，可点击已保存电脑
旁的编辑按钮，设置“办公 Mac”“家用 Mac”等别名。别名只保存在当前 Controller，
不会修改 Mac 名称、设备身份或配对授权。

## 连接失败时

- 如果 Mac 显示授权未完成，请先完成两项权限设置，再重新连接。
- 确认 Mac 上的 Roammand 正在运行，并且 Host 显示为可用。
- 如果已有另一个 Controller 正在连接或恢复，请先在 Host 本机停止该会话。
- 如果 Host 在配对后更换了 signaling 服务，请通过新服务重新配对。Controller
  保存的记录会有意保留配对时经过认证的 signaling 地址。
- 尝试让其中一台设备切换网络。没有 TURN 时，即使两台设备都能连接 signaling，
  严格网络仍可能阻止直连。
- 如果出现非预期的 Controller，请立即在 Mac 本机停止会话。

## 停止或撤销访问

移动 Controller 关闭会话时会释放所有按住的输入；App 进入后台时会关闭会话，
不会自动重连。Host 上的“停止”或“紧急停止”会终止当前会话，但保留已有授权。
如需阻止以后连接，必须在“**此电脑**”中撤销该 Controller。只在 Controller
一侧删除 Host 卡片，不会撤销 Host 端授权。

Roammand 不需要账号，但 signaling 与 STUN 服务仍会看到运行服务所必需的有限
网络及路由元数据。信任边界、元数据与诊断策略见[安全指南](../security/README.zh-CN.md)。

## 从 macOS 卸载

打开“**设置 → 高级 → 卸载 Roammand**”。受保护卸载器会移除 App、Host 组件、
launchd 配置、运行文件、日志、设备身份、配对授权、已保存 Host、偏好设置、缓存，
以及仅属于 Roammand 的屏幕录制和辅助功能授权。重新安装后会生成新的设备身份，
并需要重新配对。

开发者和源码构建者请使用[构建指南](../BUILDING.zh-CN.md)。
